## Task #16: Add Brevo Email Provider

### Migration `019_add_brevo_email_provider.sql`
```sql
CREATE TABLE IF NOT EXISTS email_provider (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    provider TEXT NOT NULL DEFAULT 'brevo',
    api_key TEXT NOT NULL,
    sender_email TEXT NOT NULL,
    sender_name TEXT NOT NULL,
    status TEXT NOT NULL DEFAULT 'unknown',    -- unknown/connected/error/suspended
    last_checked_at DATETIME,
    last_error TEXT NOT NULL DEFAULT '',
    consecutive_failures INTEGER NOT NULL DEFAULT 0,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```
- Singleton table (one row), admin-only — NOT per-user like Cloudinary
- Named `email_provider` (not `brevo_credentials`) for future flexibility

**File:** `api/internal/db/migrations/019_add_brevo_email_provider.sql`

### Backend: `brevo_handlers.go`
New file with 4 admin-only handlers:

| Endpoint | Handler | Description |
|----------|---------|-------------|
| `GET /api/admin/settings/email` | HandleGetEmailProvider | Get current config (mask API key) |
| `POST /api/admin/settings/email` | HandleSaveEmailProvider | Upsert config, auto-test on save |
| `DELETE /api/admin/settings/email` | HandleDeleteEmailProvider | Remove config |
| `POST /api/admin/settings/email/test` | HandleTestEmailProvider | Manual test connection |

- Test connection: `GET https://api.brevo.com/v3/account` with header `api-key: KEY`
- On save: auto-test and set status to connected/error
- Manual test button: same test, updates status/last_checked_at/last_error

### Health Monitoring (daily auto-check)
- Add a background goroutine in `main.go` that runs every 24 hours
- Calls `GET https://api.brevo.com/v3/account` to verify API key still valid
- Updates `status`, `last_checked_at`, `last_error`, `consecutive_failures`
- After 5 consecutive failures: set status to `suspended`
- On successful check: reset consecutive_failures to 0, status to `connected`
- Goroutine starts on server boot, uses `time.Ticker` with context cancellation

**Files:** `api/internal/api/brevo_handlers.go`, `api/cmd/api/main.go`

### Frontend: Brevo Settings in Admin Page
Add a new section/tab to the Admin page (not a separate page):

- **Email Provider card** in Admin page with:
  - API Key input (masked), Sender Name, Sender Email
  - Save / Delete buttons
  - **Test Connection** button — manual trigger
  - Status badge: Connected (green) / Error (red) / Unknown (gray) / Suspended (orange)
  - Last checked timestamp
  - Last error message (if any)
  - Info text: "Email provider is checked daily. Use Test Connection to verify manually."

**Files:** `web/src/routes/Admin.tsx`, `web/src/lib/api.ts`
